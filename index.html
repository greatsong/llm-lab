토큰화 핵심
글자 단위: 각 글자를 하나의 토큰으로 분리합니다. 사전 크기는 작지만 토큰 수가 많아집니다.
사전 크기 비교
글자 단위: 한글 11,172자 필요  /  단어 단위: 10만+ 단어 필요  /  BPE: ~32,000 토큰이면 충분!
문제 -> 아이디어 -> 알고리즘
문제: 컴퓨터에게 텍스트를 어떻게 입력할까?
아이디어: 텍스트를 작은 조각(토큰)으로 나누어 숫자로 변환하자
알고리즘: BPE -- 자주 등장하는 글자 조합을 하나의 토큰으로 병합
표현 방식
3D 임베딩 공간 -- 드래그하여 회전, 스크롤하여 확대
유사도 (단어를 클릭하세요)
임베딩 핵심
임베딩 공간에서는 의미가 비슷한 단어끼리 가까이 모입니다. 단어를 클릭하면 다른 단어와의 유사도를 확인할 수 있습니다. 임베딩 행렬도 가중치이며, 역전파로 학습됩니다.
문제 -> 아이디어 -> 알고리즘
문제: 숫자로 바꾼 토큰에 의미를 어떻게 담을까?
아이디어: 비슷한 뜻의 단어를 가까운 위치에 배치하자
알고리즘: 임베딩 -- 각 토큰을 고차원 벡터로 변환하는 학습 가능한 행렬
문장 선택
어텐션 히트맵 -- 행(가로줄)을 클릭하면 해석이 나타납니다
어텐션 해석
행(단어)을 클릭하면 해당 단어가 어디에 집중하는지 볼 수 있습니다.
어텐션 핵심
어텐션 = 각 단어가 문맥을 이해하기 위해 다른 단어에 부여하는 중요도. 색이 진할수록 더 많이 집중합니다. 어텐션 가중치도 학습으로 최적화됩니다.
문제 -> 아이디어 -> 알고리즘
문제: 문장에서 어떤 단어가 서로 관련 있는지 어떻게 알까?
아이디어: 각 단어가 다른 단어에 "주의(Attention)"를 기울이게 하자
알고리즘: Self-Attention -- Query, Key, Value로 단어 간 관계를 계산
시작 문장
나는 오늘
Temperature
1.0
보통 -- 적당히 다양한 선택
다음 단어 후보 -- 클릭하면 선택됩니다 (스텝 1/4)
LLM 작동 원리
LLM = 어텐션으로 문맥을 파악한 뒤, 다음에 올 단어의 확률을 예측하는 기계. Temperature가 낮으면 가장 높은 확률의 단어만 선택하고, 높으면 다양한 단어가 선택됩니다.
문제 -> 아이디어 -> 알고리즘
문제: 컴퓨터가 사람처럼 글을 쓸 수 있을까?
아이디어: 지금까지의 문맥을 보고 다음 단어를 확률적으로 예측하자
알고리즘: 토큰화 -> 임베딩 -> 어텐션 -> 확률 분포 -> 샘플링 = GPT
